# Project Structure

This is a **Next.js 15** portfolio application using the **App Router** architecture with TypeScript and Tailwind CSS.

## Directory Structure

```
Portfolio/
├── app/                          # Next.js App Router directory
│   ├── globals.css              # Global styles with Tailwind CSS
│   ├── layout.tsx               # Root layout component (SSR)
│   └── page.tsx                 # Home page (Client Component)
│
├── components/                   # React components
│   ├── theme-provider.tsx       # Theme provider wrapper
│   └── ui/                      # Shadcn/ui component library
│       ├── accordion.tsx
│       ├── alert-dialog.tsx
│       ├── button.tsx
│       ├── card.tsx
│       ├── dialog.tsx
│       └── ... (40+ UI components)
│
├── hooks/                       # Custom React hooks
│   ├── use-mobile.ts           # Mobile detection hook
│   └── use-toast.ts            # Toast notification hook
│
├── lib/                        # Utility functions
│   └── utils.ts                # Helper functions (cn for className merging)
│
├── public/                     # Static assets
│
├── styles/                     # Additional stylesheets
│
├── docs/                       # Project documentation
│
├── components.json             # Shadcn/ui configuration
├── next.config.mjs            # Next.js configuration
├── tsconfig.json              # TypeScript configuration
├── postcss.config.mjs         # PostCSS configuration
└── package.json               # Dependencies and scripts
```

## Key Architectural Patterns

### App Router (Next.js 15)

The project uses Next.js 15's App Router, which provides:

- File-based routing in the `app/` directory
- Server Components by default
- Nested layouts and loading states
- Improved performance and bundle size

**Key Files:**

- **`app/layout.tsx`**: Root layout (Server Component by default)
- **`app/page.tsx`**: Home page marked with `"use client"` for interactivity
- **`app/globals.css`**: Global styles and Tailwind imports

### Component Organization

#### UI Components (`components/ui/`)

- Built with **Radix UI** primitives via **Shadcn/ui**
- Fully accessible and customizable
- Consistent styling with Tailwind CSS
- Includes 40+ production-ready components

**Example components:**

```
components/ui/
├── accordion.tsx       # Collapsible sections
├── button.tsx         # Button variants
├── card.tsx           # Card containers
├── dialog.tsx         # Modal dialogs
├── form.tsx           # Form components
├── input.tsx          # Text inputs
├── select.tsx         # Dropdown selects
└── ...
```

#### Theme Provider (`components/theme-provider.tsx`)

```tsx
"use client";

import * as React from "react";
import {
  ThemeProvider as NextThemesProvider,
  type ThemeProviderProps,
} from "next-themes";

export function ThemeProvider({ children, ...props }: ThemeProviderProps) {
  return <NextThemesProvider {...props}>{children}</NextThemesProvider>;
}
```

Wraps the `next-themes` library for easy dark/light mode switching.

#### Custom Hooks (`hooks/`)

- **`use-mobile.ts`**: Detects mobile viewport
- **`use-toast.ts`**: Toast notification management

### Styling Strategy

#### Tailwind CSS 4.x

```css
/* app/globals.css */
@import "tailwindcss";
@import "tw-animate-css";

:root {
  --background: oklch(1 0 0);
  --foreground: oklch(0.145 0 0);
  /* ... more CSS variables */
}

.dark {
  --background: oklch(0.145 0 0);
  --foreground: oklch(0.985 0 0);
  /* ... dark mode overrides */
}
```

**Features:**

- **OKLCH Color Space**: Modern, perceptually uniform colors
- **CSS Variables**: Easy theming and customization
- **Dark Mode**: Built-in with CSS class toggle
- **Animations**: tw-animate-css for smooth transitions

#### Utility Functions (`lib/utils.ts`)

```typescript
import { clsx, type ClassValue } from "clsx";
import { twMerge } from "tailwind-merge";

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs));
}
```

The `cn` function intelligently merges Tailwind classes, resolving conflicts.

#### `tsconfig.json`

```json
{
  "compilerOptions": {
    "lib": ["dom", "dom.iterable", "esnext"],
    "target": "ES6",
    "jsx": "preserve",
    "paths": {
      "@/*": ["./*"]
    }
  }
}
```

**Path Aliases**: `@/*` maps to the root directory for clean imports:

```tsx
import { Button } from "@/components/ui/button";
import { cn } from "@/lib/utils";
```

#### `components.json` (Shadcn/ui)

```json
{
```

Configures Shadcn/ui component generation and styling.

## File Naming Conventions

- **Components**: `kebab-case.tsx` (e.g., `theme-provider.tsx`)
- **Pages**: `page.tsx` (App Router convention)
- **Layouts**: `layout.tsx` (App Router convention)
- **Utilities**: `kebab-case.ts` (e.g., `use-mobile.ts`)
- **Config**: `kebab-case.extension` (e.g., `next.config.mjs`)

## Server vs Client Components

### Server Components (Default)

```tsx
// app/layout.tsx - No "use client" directive
export default function RootLayout({ children }) {
  return <html>{children}</html>;
}
```

**Benefits:**

- Zero JavaScript sent to client
- Direct database access
- Server-side data fetching
- SEO-friendly

### Client Components

```tsx
// app/page.tsx
"use client"; // Required for interactivity

import { useState } from "react";

export default function Page() {
  const [count, setCount] = useState(0);
  return <button onClick={() => setCount(count + 1)}>{count}</button>;
}
```

**Required for:**

- React hooks (`useState`, `useEffect`, etc.)
- Event handlers
- Browser APIs
- Third-party libraries with client-side dependencies
